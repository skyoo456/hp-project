# 기능 제안·정리 (HOMEPLATE)

개발자 관점에서 **바꾸면 좋겠다 / 없애도 좋겠다 / 추가했으면 한다** 는 제안과, **애매한 기능**을 정리했습니다.

---

## 1. 추가하면 좋은 기능

| 제안                         | 설명                                                                                              |
| ---------------------------- | ------------------------------------------------------------------------------------------------- |
| **대기열(Queue) 페이지**     | 예매 오픈 시 대기실. "대기 순번", "예상 대기 시간" 표시. 이미 별도 페이지로 추가 예정.            |
| **검색**                     | 헤더 검색 아이콘 → 팀/경기/구장 검색 페이지 또는 모달. 현재는 "준비중" 알림만 있음.               |
| **알림(공지)**               | 예매 오픈 알림, 경기 D-1 알림 등. 백엔드 푸시/이메일 연동 시 좋음.                                |
| **좌석 맵 실제 데이터 연동** | 현재 좌석은 더미 레이아웃. 백엔드에서 구역별 좌석 수·매진 여부 받아서 표시하면 좋음.              |
| **굿즈 상세/장바구니**       | 굿즈는 목록만 있고 "상세" 클릭 시 알림만 있음. 상세 페이지 + 장바구니 플로우 추가 시 완성도 상승. |
| **결제 연동**                | 결제는 "더미"로 티켓만 발급. 실제 PG/결제창 연동 시 결제 취소·환불 플로우도 정리 필요.            |

---

## 2. 없애거나 단순화해도 좋은 것

| 제안                      | 설명                                                                                                      |
| ------------------------- | --------------------------------------------------------------------------------------------------------- |
| **dev/map-editor**        | 구역 맵 편집용. 운영에는 불필요할 수 있으므로, 개발용으로만 두거나 보호(비공개/권한) 후 유지.             |
| **일정 페이지 "달력" 뷰** | 리스트만으로도 충분하다면 달력 뷰는 선택. 유지 시 "해당일 첫 경기"만 링크되는 현재 동작을 그대로 두면 됨. |
| **순위·기록 "팀 필터"**   | 전체 팀만 쓴다면 필터 제거해도 됨. 팀별로 보고 싶다면 유지.                                               |

---

## 3. 애매한 기능 (정리 필요)

| 기능                     | 애매한 점                                                                                                                 | 제안                                                                               |
| ------------------------ | ------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------- |
| **로그인/회원가입**      | 현재 더미(이메일+비밀번호만 체크). 실제 인증 주체(자체 DB vs 소셜 vs 둘 다)가 정해지면 auth API·store 연동 방향이 달라짐. | 인증 방식 확정 후 `shared/api/auth.ts`, `features/auth/store.ts` 에만 반영하면 됨. |
| **예매 "공유" 버튼**     | 경기 상세의 "공유"가 현재 준비중.                                                                                         | URL 공유만 할지, SNS 공유까지 할지 정한 뒤 구현.                                   |
| **고객센터/이용약관 등** | 푸터 링크가 "준비중" 알림.                                                                                                | 백엔드 없이 정적 페이지로 만들어도 됨.                                             |
| **예매 오픈 시간**       | 관리자에서 "예매 오픈 일시" 설정 가능. 실제 오픈 시 대기열·티켓팅 플로우와 맞출지 정해야 함.                              | 백엔드와 함께 "오픈 시점에만 예매 가능" 등 규칙 정리.                              |
| **자동 종료**            | 경기 시작 24시간 전부터 "종료" 표시하는 옵션. 실제 경기 취소·연기와 정책이 다를 수 있음.                                  | 운영 정책에 맞게 "종료" 조건만 조정하면 됨.                                        |

---

## 4. 정리 요약

- **추가 권장**: 대기열 페이지(진행 예정), 검색, 알림, 좌석 실데이터, 굿즈 상세/장바구니, 결제 연동.
- **제거/단순화 검토**: map-editor 운영 노출, 일정 달력/순위 필터.
- **정리 필요**: 로그인 방식, 공유 범위, 푸터 정책 페이지, 예매 오픈/자동 종료 정책.

이 문서는 나중에 백엔드/기획과 맞춰가면서 업데이트하면 됩니다.
